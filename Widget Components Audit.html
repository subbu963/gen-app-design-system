Issue 1 · tone tints bypass the token vocabulary

Card and Stack accept a tone prop with seven values: accent / positive / warning / danger / warm / cool / night. The styling lands in primitives.css as inline RGBA gradients. Three of those values (warm, cool, night) are new vocabulary — they have no presence in our design tokens. The other four mimic our existing tints (--color-accent-tint, --tint-green, --tint-orange, --tint-red) but use slightly different opacity values and a radial-gradient origin we never spec'd.

Recommended fix

Add seven tone tokens to colors_and_type.css and rewrite primitives.css to consume them. This (a) gets the colors into the system's source of truth, (b) lets the light theme override them later, (c) survives any future palette tweak without touching widget-components. 

/* colors_and_type.css — add to :root + :root[data-theme="light"] */
--tone-accent:   radial-gradient(circle at 80% 0%, var(--color-accent-tint), transparent 60%);
--tone-positive: radial-gradient(circle at 80% 0%, var(--tint-green),  transparent 60%);
--tone-warning:  radial-gradient(circle at 80% 0%, var(--tint-orange), transparent 60%);
--tone-danger:   radial-gradient(circle at 70% 0%, var(--tint-red),    transparent 65%);
--tone-warm:     radial-gradient(circle at 80% 0%, rgba(255,159,10,0.22), transparent 62%);
--tone-cool:     linear-gradient(180deg,  rgba(64,128,200,0.20),  transparent 70%);
--tone-night:    linear-gradient(180deg,  rgba(94,92,230,0.22),   transparent 65%);

/* libs/widget-components/src/primitives.css — replace each rule */
.wc-tone-accent   { background-image: var(--tone-accent); }
.wc-tone-positive { background-image: var(--tone-positive); }
/* … etc */
Bonus: consider renaming warm / cool / night to something that signals their intent — these aren't tones, they're mood backdrops for weather and time-of-day. Could be a separate backdrop prop on Card if you want the semantics clean.

Stepper · RangeSlider · ProgressRing

Stepper: --bg-input shell, mono value, +/− on either side. RangeSlider: two overlapping native ranges with shared track + accent fill between the handles. ProgressRing: same accent/track contract as Progress but circular.

Banner

Four tones — info / success / warning / error — each on its matching tint (accent-tint, tint-green, tint-orange, tint-red). Title + supporting text.

Use as an in-widget status row. Distinct from the budget/gate footer in Settings (which is also red-tint but uses the row pattern, not a banner).

Skeleton · Spinner

Skeleton has three variants — text (n lines, last 60% wide), circle (40×40), block (full × 64px). Spinner is 18px default, accent-on-elevated ring.

Both share the shimmer keyframes and the same --bg-card-elevated → --bg-hover → --bg-card-elevated gradient — a single visual language for "we're working on it".
