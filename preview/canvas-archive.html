Pattern notes
  • Three groups, three colors. Green for actions that proceed silently; default for no-ops; orange for anything that requires user action. Red would be "blocking" — used only if a widget can't be rehydrated at all.
  • Conflict resolution is a footnote, not a modal. "A canvas named X already exists" sits as an inline orange info row — never a blocking dialog.
  • Re-auth is the per-row action. The import doesn't wait — it proceeds and leaves the MCP server in the "re-auth needed" state from the Extensions list. The user can deal with it later.
  • Export menu lives under a divider. Session ops above, archive ops below — the dock menu shouldn't grow indefinitely.
